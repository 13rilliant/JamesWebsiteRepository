James Attempts to Explain Relative Linking

There are essentially three parts to this resource:

  1. Reading what I've written on the website
  2. Interacting with the links to understand how the user sees them
  3. Looking at the pictures I've attached which explain folder structure

It's important to remember that when linking files you've written for a website together, you are always linking in a relative way. This means that you must format the links in your html to treat the page you're currently focused on as the root of all other pages. If you need to go a folder up, indicate that in your link. If you need to target a file a folder down, indicate that in your link.

The server doesn't read links the same way your computer does. Your computer can read a link address all the way back from the C drive, since that's the system level world it knows. Similarly, the server only knows about the files you've uploaded. This means that you This is why it's so important to link only based on the folders and files you've uploaded. When editing on your computer, it's best to pretend only the files you're editing with exist, since that's all a server will be able to judge from later.

Our First Link ← working link

In html, the link looks like this: 

<a href="link1.html">Our First Link</a>

This link works because the file it's targeting is located in the same folder as the file (or page) we're currently on. You can see it was written with an href tag that doesn't have any location indicators, only a file name.

Our Second Link ← broken link

In html, the link looks like this: 

<a href="link2.html">Our Second Link</a>

This link is written just like "Our First Link", but it doesn't work because it fails to indicate that the file we're looking for is in a different folder from the file (page) we're currently on. In this case, "link2.html" is in a folder one below "index.html".

If you want to reach link2.html, you must tell the server to seek one folder down. In this case, the new format of your link inside an anchor ("a") tag would look like: 

<a href="Folder/link2.html">Woot woot send for the loot</a>
